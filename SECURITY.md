# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **do not** open a public issue. Doing so exposes the platform and its users to immediate risk.

Please report all security vulnerabilities privately to the repository owner by emailing [priyanshupriyesh@gmail.com](mailto:priyanshupriyesh@gmail.com) with the subject line `SECURITY VULNERABILITY REPORT`.

### Please include the following details in your report:

1. **Description of the vulnerability** and its potential impact.
2. **Steps to reproduce** the issue (exploit code, screenshots, or videos are highly appreciated).
3. **Environment details** (OS, Node.js version, browser, etc.) if applicable.

Once received, the issue will be investigated privately, and a patch will be prepared and deployed before public disclosure.

## Supported Versions

Only the latest active release on the `main` branch is supported for security updates and patches.
